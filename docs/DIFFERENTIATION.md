# TTS Differentiation

## 与 Bob / Easydict 的差异

Bob 和 Easydict 是成熟的 macOS 翻译工具，覆盖词典、划词、输入翻译、多服务商、多结果展示和更广泛的语言学习场景。TTS 不试图复制这些产品的完整能力，也不把目标设定为替代所有翻译工具。

TTS 的方向更窄：它专注于轻量化 AI 截图翻译，优先服务“看到屏幕内容后快速 OCR、清理文本、交给 AI 翻译并得到可读结果”的流程。相比传统词典式体验，TTS 更关注截图识别后的上下文恢复、AI 模型质量和失败时的自动恢复。

## 核心优势

- 轻量：TTS 是菜单栏工具，默认不显示主窗口，围绕快捷键和悬浮面板完成高频操作。
- AI-first：TTS 优先面向 AI 翻译质量，而不是词典条目、释义列表或多译文比较。
- OCR 后处理：TTS 将截图 OCR 视为核心流程的一部分，重视段落恢复、自然拼接和可复制文本。
- 模型路由：TTS 保留多服务商配置、默认服务商和优先级，为后续更智能的模型选择打基础。
- fallback：TTS 在服务失败、网络异常、认证问题或限流时，应优先给出可读错误并尝试可控 fallback。
- 原生 macOS 体验：TTS 使用 Swift、SwiftUI、AppKit、Vision 和 Keychain，保持轻量、本地、贴近系统的交互方式。

## 不追求大而全

TTS 不追求成为全功能词典、插件市场或大型语言学习平台。后续功能必须服务于轻量、OCR、AI 翻译质量、模型路由、fallback 和 macOS 原生体验这条主线。

当一个功能会显著增加配置复杂度、维护成本或产品认知负担时，应优先判断它是否强化核心截图翻译流程。不能强化核心流程的功能，应延后或不做。
