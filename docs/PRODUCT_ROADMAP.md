# TTS Product Roadmap

## 产品定位

TTS 是一个轻量、开源、AI 翻译优先的 macOS 菜单栏截图翻译工具。它面向需要快速理解屏幕内容的用户，优先解决截图 OCR、OCR 文本后处理、AI 翻译质量、模型路由和服务 fallback 这些核心问题。

TTS 的体验目标是：少打扰、响应快、配置清晰、原生贴合 macOS。应用应保持菜单栏工具的轻量形态，不演变成复杂的词典、知识库或插件平台。

## 不做什么

- 不做传统词典式翻译，不以词条、释义、例句、发音、词形变化为核心。
- 不做多翻译结果并排展示，不把用户注意力分散到大量候选译文比较中。
- 不做复杂插件市场，不引入需要长期维护的第三方扩展生态。
- 不做大而全的语言学习工具，不承担背单词、笔记管理、阅读器等场景。
- 不把设置和工作流做成重型产品，优先保持可理解、可恢复、可维护。

## 阶段路线

### 阶段 1：稳定轻量 MVP

- 稳定划词翻译、输入翻译、截图翻译、截图 OCR 和静默 OCR。
- 保持菜单栏常驻、不抢焦点、快捷键优先的 macOS 原生体验。
- 确保历史记录、收藏夹、权限引导和 Keychain 密钥保存可靠。
- 保持所有网络错误和配置错误对用户可读。

### 阶段 2：AI 翻译质量模式

- 围绕 AI 翻译质量优化默认提示词、目标语言表达和上下文处理。
- 为截图 OCR 翻译建立更清晰的前处理和后处理策略。
- 区分速度优先与质量优先的使用场景，但避免复杂模式堆叠。

### 阶段 3：模型路由与 fallback

- 完善多服务商优先级、默认服务商和认证失败 fallback 行为。
- 让用户能清楚理解当前使用的服务商、失败原因和切换结果。
- 在不增加复杂依赖的前提下，提升服务可用性和恢复能力。

### 阶段 4：原生体验打磨

- 继续优化悬浮面板、OCR 结果面板、设置页和菜单栏图标。
- 同时适配浅色模式和深色模式。
- 保持小尺寸、低干扰、可快速关闭的菜单栏工具体验。

## 近期优先级

- 明确产品边界，避免向传统词典或大而全翻译平台扩张。
- 改进截图 OCR 到 AI 翻译之间的文本清理、段落恢复和错误反馈。
- 优化服务商配置、模型选择、fallback 状态和用户可读错误。
- 打磨设置页与悬浮面板的深浅色模式一致性。
- 保持 `swift build` 稳定通过，避免引入不必要依赖。

## MVP 标准

- 用户可以通过快捷键完成划词翻译、输入翻译、截图翻译、截图 OCR 和静默 OCR。
- 截图 OCR 能输出可复制、可翻译、段落基本可读的文本。
- 至少一个默认服务商无需复杂配置即可完成测试翻译。
- API Key 等敏感信息只保存在 Keychain，不进入仓库和配置文件。
- 权限缺失、网络失败、认证失败和无识别文本时都有清晰提示。
- 应用保持菜单栏工具形态，不默认显示主窗口，不打断当前工作流。
- README 和产品文档能清楚说明 TTS 的定位、边界和差异化方向。
