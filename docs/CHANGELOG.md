# TTS 修改记录

本文件用于记录 TTS 的产品、UI、配置和核心能力变更，方便后续回看设计取舍与开发进度。

## 2026-04-27

### 翻译服务商与模型选择

- 暂停任务 7-9 的模型 Profile、TranslationRouter 和场景路由实现，当前阶段不引入自动模型路由，翻译流程继续按默认服务商和 fallback 优先级执行。
- 保留设置页“模型配置”作为当前服务商模型编辑入口，不接入真实路由逻辑。
- 新增 DeepSeek 和 Gemini 大模型服务商，并继续保留 OpenAI 兼容接口、智谱 GLM 和硅基流动。
- 在翻译服务设置中将 DeepSeek 和 Gemini 归入“AI 大模型”分组。
- 为智谱 GLM、硅基流动、DeepSeek、Gemini 和 OpenAI 兼容接口提供常用模型下拉选择，同时仍允许用户手填自定义模型。
- 旧配置中的服务商显示名会自动归一到当前名称，例如将 GLM-4-Flash 显示为“智谱 GLM”。

### PromptBuilder 与 AI 模式

- 新增独立 `PromptBuilder`，集中生成大模型类 provider 使用的 system/user prompt。
- `PromptBuilder` 只依赖 `TranslationMode`、源文本和目标语言，不重新引入模型路由、偏好评分或复杂配置。
- 为快速翻译、准确翻译、自然表达、学术翻译、技术翻译、OCR 修复、双语输出和翻译润色分别提供独立 prompt。
- 强化学术翻译中的正式语体、限定条件和逻辑关系保留要求。
- 强化技术翻译中的代码块、变量名、API 名称、命令行内容和 Markdown 结构保留要求。
- 明确 OCR 修复模式不是翻译，只做原语言文本清理和段落恢复。
- 调整悬浮窗口 AI 模式说明区，保留右侧解释，并支持在模式菜单中悬停预览时即时更新说明，再点击确认重译。
- 优化悬浮窗口钉住反馈，鼠标悬停钉住按钮时显示轻量提示气泡，同时保留按钮状态动画。

### OCR 文本后处理

- 新增 `OCRTextPostProcessor`，对 OCR 原始文本做本地规则清理，不调用 AI，也不做翻译。
- 支持 `plainText`、`article`、`code`、`markdown` 和 `auto` 五种处理模式，其中 `auto` 会根据文本特征判断代码、Markdown 或普通文章。
- 后处理会清理多余空行、修复英文断词连字符换行、合并同段短行、修复中英文不自然空格，并尽量保留 URL、数字单位、项目符号和代码结构。
- 截图 OCR、静默 OCR 和截图翻译现在默认使用处理后文本，提升后续翻译输入质量。
- OCR 结果面板默认展示处理后文本，并提供查看原始 OCR 文本入口；复制按钮默认复制处理后文本。

### OCR AI 修复

- 在 OCR 结果面板新增 `AI 修复` 按钮，点击后通过 `TranslationMode.ocrCleanup` 调用大模型修复 OCR 文本，但不做翻译。
- AI 修复结果会保留原语言，重点处理 OCR 错字、错误换行和段落恢复，同时尽量保留代码、专有名词、数字和 URL。
- 修复完成后，OCR 面板支持在“处理后文本 / AI 修复 / 原始 OCR”之间切换查看，并可直接复制 AI 修复文本。
- 新增 `继续翻译` 入口，优先使用 AI 修复后的文本继续走截图翻译流程。
- 当默认服务不支持大模型 prompt 时，会提示用户切换到 AI 大模型服务，不再将 OCR 修复错误回退到传统翻译服务。

### Fallback 简化

- 将 fallback 收敛为轻量方案：默认始终先使用当前服务和模型，请求失败后最多尝试 3 次，不再按所有已启用服务做优先级遍历。
- 设置页新增全局 fallback 设置，只保留自动 fallback 开关、备用服务商和备用模型，不再暴露优先级和按认证失败单独切换的配置。
- `timeout` 会先重试一次当前服务，再尝试备用服务；`invalidAPIKey` 不会重复尝试同一个服务；`rateLimited` 会优先切换备用服务。
- 最终失败提示改为简洁用户文案，详细尝试链路仅保留在 debug 日志中，且不包含 API Key。

### 图片文件 OCR

- 菜单栏新增 `图片文件 OCR...` 入口，支持从本地选择 `png`、`jpg`、`jpeg`、`webp` 和 `heic` 图片文件进行 OCR。
- 图片文件 OCR 复用现有 `OCRService`、`OCRTextPostProcessor` 和 `OCRResultPanel`，结果同样支持复制文本、AI 修复和继续翻译。
- 图片文件 OCR 不依赖屏幕录制权限；设置页权限说明已区分截图 OCR 与图片文件 OCR。

### 图片覆盖翻译模式

- 在 `TranslationMode` 中新增 `imageOverlay`，设置页和相关 UI 中显示为“图片覆盖翻译”。
- 为 `imageOverlay` 增加专用 prompt，强调简洁输出、不解释、不添加前缀，并保留数字、品牌名、代码和 URL。
- 当前只补充模式和 prompt 能力，不改变普通截图翻译默认使用的模式，为后续截图翻译覆盖功能预留接入点。

### OCR 文本块坐标识别

- `OCRService` 新增基于 `NSImage` 的 `recognizeTextBlocks` 能力，除了纯文本外也可以返回每个 OCR 文本块的内容、置信度和坐标。
- OCR 文本块的 `boundingBox` 已从 Vision 归一化坐标转换为图片坐标系，便于后续做截图翻译覆盖绘制。
- 现有纯文本 OCR、OCR 结果面板和截图翻译流程保持不变，只是在底层补充了文本块定位数据。

### OCR 文本块合并

- 新增 `OCRTextBlockGrouper`，用于将 Vision OCR 返回的碎片化小块合并成更适合翻译和覆盖显示的文本块。
- 合并器会优先合并同一行中距离较近的片段，并在普通段落场景下适度做纵向合并；列表项、代码样式文本和跨度过大的区域会尽量保持独立。
- 合并后的文本块会保留原始阅读顺序，并将 `boundingBox` 扩展为覆盖所有子块区域。

### 截图翻译覆盖渲染器

- 新增 `ScreenshotTranslationOverlayRenderer`，可根据原始截图、OCR 文本块和对应译文生成带中文覆盖层的新图片。
- 第一版支持 `solid`、`translucent` 和 `bubble` 三种覆盖样式，统一在文本块区域上绘制背景框并在框内排版中文译文。
- 渲染器会根据区域大小自动收缩字号、在需要时换行，并尽量将覆盖内容限制在图片边界内。

### 截图翻译覆盖入口

- 菜单栏新增 `截图翻译覆盖` 入口，走独立流程，不替代现有截图翻译和截图 OCR。
- 新流程会在截图后读取原始 `NSImage`，识别 OCR 文本块，做文本块合并，并以 `TranslationMode.imageOverlay` 对每个块逐个翻译。
- 单个文本块翻译失败时会保留原文继续生成预览图；整体翻译失败时会给出简洁错误提示。
- 生成完成后会打开单独的预览窗口，展示带中文覆盖层的截图结果。

### 翻译图片预览窗口

- 新增 `TranslatedImagePreviewWindow`，用于展示截图翻译覆盖结果，并提供复制图片、保存 PNG、切换覆盖样式、重新生成和关闭操作。
- 预览窗口支持缩放查看，切换样式或点击重新生成时会重新调用 `ScreenshotTranslationOverlayRenderer`。
- 保存操作使用 `NSSavePanel`，复制图片会直接写入系统剪贴板，窗口外观继续跟随系统深浅色模式。

### 设置与交互收敛

- 为 `截图翻译覆盖` 补充全局快捷键入口，并接入快捷键设置页。
- 删除设置页“通用”中的“产品定位”展示，避免无效信息占位。
- 删除独立的“模型配置”页，相关模型与 endpoint 配置继续保留在“翻译服务”页中，减少重复入口。
- 优化悬浮截图翻译面板中的 AI 模式控件布局，避免“AI 模式”标题被挤成两行。
- 调整截图翻译覆盖预览的默认缩放逻辑，优先让整张图片完整出现在窗口内，只有主动放大后才需要明显滚动查看。
- 轻微提高 OCR 准确模式的识别灵敏度，增强对更小文字区域的捕捉能力。

### 覆盖翻译体验修复

- 修复截图翻译中切换 AI 翻译风格后未自动展开版本对比的问题，重译成功后会再次显示上一版与当前版。
- 将截图翻译覆盖的默认样式改为 `solid` 纯色覆盖。
- 截图翻译覆盖生成过程中改为持续显示带转圈的加载提示，直到预览窗口打开或流程失败为止。

### 模型建议补全

- 将 AI 服务商的模型建议从纯字符串升级为“显示名 + 实际模型值”，便于展示版本信息，同时仍向 API 发送正确的模型编码。
- 补全 OpenAI 兼容接口、智谱 GLM、Gemini、DeepSeek、硅基流动的常用建议模型列表。
- DeepSeek 下拉中明确显示 `deepseek-chat` 与 `deepseek-reasoner` 对应的 DeepSeek-V3.2 语义，避免只看到 API 别名而难以理解版本。

## 2026-04-26

### 产品定位与文档

- 新增 `docs/PRODUCT_ROADMAP.md`，明确 TTS 的产品定位、不做什么、阶段路线、近期优先级和 MVP 标准。
- 新增 `docs/DIFFERENTIATION.md`，说明 TTS 与 Bob / Easydict 的差异，以及轻量、AI-first、OCR 后处理、模型路由和 fallback 的核心方向。
- 更新 `README.md` 顶部简介，强调 TTS 专注轻量化 AI 截图翻译，而不是传统词典式翻译。

### macOS 外观适配

- 移除强制浅色模式逻辑，让应用跟随 macOS 系统深色/浅色外观。
- 调整主要 UI 使用系统颜色，提升深色模式和浅色模式下的可读性。

### 悬浮翻译窗口

- 修复截图翻译加载期间移动窗口后，结果出现时窗口回到初始位置的问题。
- 修复加载期间关闭悬浮窗口后，翻译完成仍重新弹出窗口的问题。
- 将悬浮翻译窗口改为更现代的 macOS 卡片式 UI，包含原文、译文、状态提示和底部操作区。
- 保持悬浮窗口不抢焦点，并继续显示在鼠标附近。
- 修复原文/译文字号不一致、原文无法滚动、底部复制和收藏按钮不可见的问题。
- 新增 AI 模式重译入口，支持在悬浮窗口中切换翻译风格后重新翻译。
- 将上一版和当前版对比从独立弹窗改为悬浮窗口内联展开，切换风格后用动画展开为左右对比布局。
- 新增钉住按钮，钉住后点击空白处不会关闭悬浮窗口。
- 增大悬浮窗口和对比模式尺寸，提高原文、译文和错误信息字号，减少窗口空洞感并提升阅读性。
- 统一悬浮窗口正文阅读字号，原文、译文、对比内容和错误信息都使用接近译文的字号，并压缩状态区留白，让布局更紧凑。
- 调整悬浮窗口内容区高度和 footer 布局优先级，确保复制、收藏和关闭按钮始终可见；同时增强对比模式中上一版和当前版的视觉区分。

### 设置页信息架构

- 将设置页重构为分区结构：通用、快捷键、翻译服务、AI 模式、模型配置、权限与隐私。
- 保留 Provider、Endpoint、Model、Target Language、API Key、快捷键和权限状态等原有设置能力。
- 统一快捷键设置页与其他设置分区的 UI 表现。
- 移除权限与隐私页中多余的“检查权限”按钮。
- API Key 继续使用 Keychain 保存，不改变原有存储方式。

### 翻译方向与服务商分组

- 将目标语言输入改为翻译方向选择，支持自动检测到中文、自动检测到英文、英文到中文、中文到英文、日文到中文、韩文到中文和自定义。
- 设置页和输入翻译窗口都使用翻译方向选择器。
- 翻译方向选择器使用系统 bordered 按钮样式，让选中项常态显示灰色按钮背景。
- 在翻译服务设置中区分 AI 大模型服务商和传统翻译服务商。

### AI 翻译模式

- 新增 `TranslationMode` 数据结构，包含快速翻译、准确翻译、自然表达、学术翻译、技术翻译、OCR 修复翻译、双语输出、翻译并润色。
- 每个 AI 翻译模式提供显示名、说明、系统 prompt 和用户 prompt 模板。
- 在配置中保存默认 AI 翻译模式，旧配置缺失时默认使用准确翻译。
- 设置页 AI 模式分区支持选择默认 AI 翻译模式。
- 将 AI 翻译模式接入大模型类 Provider 请求构造，包括 OpenAI-compatible、GLM 和 SiliconFlow。
- 传统翻译服务商保持原有行为，不强行使用 prompt。
- 翻译历史记录本次使用的 AI 翻译模式，旧历史记录兼容默认准确翻译。
- 优化学术翻译 prompt，使其更偏论文/报告风格，强调术语一致、逻辑清晰、客观语气和不添加原文外信息。

### 工作流约定

- 后续开发默认直接在 `main` 上修改。
- 每次修改后继续运行 `swift build`，并检查 `git diff` 与 `git status`。
- 重要产品、UI、配置和行为变更继续追加记录到本文档。
