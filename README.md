# TTS

#### 一个轻量、快速、好用的 macOS 截图翻译与 OCR 工具

## 介绍

TTS 是一个常驻菜单栏的 macOS 翻译工具，支持划词翻译、输入翻译、截图翻译、截图 OCR、图片 OCR 和截图译文覆盖。

它适合这些场景：

- 看英文网页、论文、文档时，快速划词翻译
- 遇到无法复制的图片、截图、软件界面时，直接截图识别并翻译
- 想把图片里的文字翻译后覆盖回原图，获得更直观的阅读体验
- 需要在不同翻译服务和 AI 模型之间自由切换
- 想要一个轻量、不打扰、贴近 macOS 原生体验的菜单栏工具

TTS 不是复杂的词典软件，也不是大而全的翻译平台。它更专注于一个核心目标：

> 让 macOS 上的划词、截图、OCR 和 AI 翻译变得更快、更自然、更顺手。

## 功能特性

### 多种翻译方式

TTS 支持多种常用翻译入口：

| 功能 | 默认快捷键 | 说明 |
| --- | --- | --- |
| 划词翻译 | `Option + D` | 选中文本后快速翻译 |
| 输入翻译 | `Option + A` | 打开输入窗口，手动输入文本翻译 |
| 截图翻译 | `Option + S` | 截取屏幕区域，识别文字并翻译 |
| 截图翻译覆盖 | `Shift + Option + A` | 截图后将译文覆盖回原图 |
| 截图 OCR | `Shift + Option + S` | 截图后只识别文字，不自动翻译 |
| 静默截图 OCR | `Option + C` | 截图识别后直接复制文字到剪贴板 |
| 图片文件 OCR | 菜单栏入口 | 选择本地图片并识别文字 |

### 截图翻译覆盖

截图翻译覆盖是 TTS 的核心特色功能。

你可以直接截取一段英文界面、网页、海报、软件窗口或图片，TTS 会识别其中的文字，并把翻译结果覆盖显示在原图上。

当前支持：

- 纯色覆盖
- 半透明覆盖
- 气泡覆盖
- 图片预览
- 复制图片
- 保存为 PNG
- 切换覆盖样式
- 重新生成
- 缩放查看

这个功能适合阅读图片、软件界面、教程截图、论文图表、网页截图等无法直接复制文字的内容。

### OCR 文字识别

TTS 支持截图 OCR 和本地图片 OCR。

OCR 结果可以：

- 直接复制
- 查看原始识别文本
- 继续翻译
- 使用 AI 修复识别结果
- 保存到历史记录

对于截图中常见的断行、多余空格、英文换行、中英文混排等问题，TTS 会尽量整理成更适合阅读和翻译的文本。

### AI 翻译模式

TTS 内置多种翻译模式，适合不同使用场景：

- 快速翻译
- 准确翻译
- 自然表达
- 学术翻译
- 技术翻译
- 双语输出
- 翻译并润色
- OCR 修复
- 图片覆盖翻译

你可以根据内容类型选择不同模式。比如看论文时使用学术翻译，看技术文档时使用技术翻译，阅读普通网页时使用自然表达或快速翻译。

### 多服务商支持

TTS 支持多种翻译服务和 AI 服务商：

- OpenAI 兼容接口
- 智谱 GLM
- 硅基流动
- DeepSeek
- Gemini
- DeepL
- Google Cloud Translation
- Microsoft Translator / Bing
- 百度翻译
- 腾讯云机器翻译 TMT
- 火山翻译
- MyMemory

不同功能可以使用不同服务商。例如：

- 划词翻译使用速度更快的模型
- 截图翻译使用更准确的模型
- OCR 修复使用更适合文本整理的 AI 模型
- 截图覆盖翻译使用单独配置的服务商

### 场景化配置

TTS 支持按使用场景配置服务商和模型。

当前可单独配置：

- 划词翻译
- 输入翻译
- 截图翻译
- OCR AI 修复
- 截图覆盖翻译

每个场景都可以设置主服务商和备用服务商。主服务不可用时，可以自动尝试备用服务，减少翻译失败的情况。

### 悬浮翻译窗口

翻译结果会显示在轻量的悬浮窗口中。

悬浮窗口支持：

- 不抢焦点显示
- 复制译文
- 收藏结果
- 钉住窗口
- 切换翻译模式后重新翻译
- 查看上一版和当前版结果

适合在浏览器、PDF、聊天软件、代码编辑器等不同应用中快速使用。

### 历史记录与收藏

TTS 会保存翻译历史，方便回看之前翻译过的内容。

支持：

- 搜索历史记录
- 复制历史内容
- 删除单条记录
- 清空历史记录
- 收藏重要翻译
- 单独查看收藏夹

删除历史记录不会自动删除收藏内容。

### 安全保存 API Key

TTS 会将 API Key 保存到 macOS Keychain 中，不会明文写入项目文件。

你可以在设置页中管理不同服务商的 API Key、Endpoint 和模型名称。

## 安装与运行

### 从源码运行

```bash
swift build
swift run
```

启动后，TTS 会出现在 macOS 右上角菜单栏。

### 打包为 macOS App

```bash
scripts/build_app.sh
```

构建完成后，产物位于：

```bash
build/Release/TTS.app
```

安装到 `/Applications`：

```bash
rm -rf /Applications/TTS.app
cp -R build/Release/TTS.app /Applications/
open /Applications/TTS.app
```

### 使用 Xcode 构建

```bash
xcodebuild -project tts.xcodeproj -scheme tts -configuration Release build
```

## 权限说明

TTS 需要以下 macOS 权限：

| 权限 | 用途 |
| --- | --- |
| 辅助功能 | 用于读取当前选中的文字 |
| 屏幕录制 | 用于截图 OCR、截图翻译和截图翻译覆盖 |
| 文件访问 | 用于选择本地图片并进行 OCR |

图片文件 OCR 不需要屏幕录制权限。

如果已经授权但应用仍提示未授权，请完全退出 TTS，然后重新打开 `/Applications/TTS.app`。macOS 会按具体 App 路径记录权限，移动 App 后可能需要重新授权。
